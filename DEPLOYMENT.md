# Cloud Daftar Deployment Guide

This app is ready to deploy as a Vercel hosted Next.js app with Neon/Postgres for data and UploadThing for user files. Do not depend on local disk for uploaded files on Vercel.

## Production Architecture

- App hosting: Vercel, Next.js App Router
- Runtime: Node.js 20.9 or newer
- Database: Neon Postgres or another managed Postgres provider
- File storage: UploadThing, already installed and wired through `/api/upload`
- Authentication: NextAuth with Prisma adapter
- Email: SMTP or Brevo SMTP
- PDFs: generated by the app. Heavy Puppeteer routes need a Vercel-compatible Chromium/browserless setup before production use.

## Before Pushing To Git

Run these locally:

```bash
npm ci
npx prisma generate
npx tsc --noEmit
npm run build
```

Confirm these are not committed:

- `.env`
- `.next/`
- `.npm-cache/`
- `storage/`
- `tools/zatca-sdk/`
- local logs such as `.next-dev.log`

## Required Vercel Environment Variables

Set these in Vercel Project Settings -> Environment Variables.

| Variable                                                                      | Required        | Notes                                                                 |
| ----------------------------------------------------------------------------- | --------------- | --------------------------------------------------------------------- |
| `DATABASE_URL`                                                                | Yes             | Neon/Postgres connection string with SSL.                             |
| `NEXTAUTH_SECRET`                                                             | Yes             | Generate a strong value, for example `openssl rand -base64 32`.       |
| `NEXTAUTH_URL`                                                                | Yes             | Production URL, for example `https://your-app.vercel.app`.            |
| `NEXT_PUBLIC_APP_URL`                                                         | Yes             | Same public production URL.                                           |
| `NEXT_PUBLIC_APP_NAME`                                                        | Yes             | Usually `Cloud Daftar`.                                               |
| `UPLOADTHING_TOKEN`                                                           | Yes             | Needed for logos, product images, payment screenshots, and documents. |
| `SUPER_ADMIN_NAME`                                                            | First bootstrap | Only needed before first production seed/bootstrap.                   |
| `SUPER_ADMIN_EMAIL`                                                           | First bootstrap | Only needed before first production seed/bootstrap.                   |
| `SUPER_ADMIN_PASSWORD`                                                        | First bootstrap | Use a strong temporary password and rotate it.                        |
| `SMTP_HOST` / `SMTP_PORT` / `SMTP_USER` / `SMTP_PASSWORD` / `SMTP_FROM`       | Optional        | Required for invites and general email.                               |
| `BREVO_SMTP_HOST` / `BREVO_SMTP_PORT` / `BREVO_SMTP_USER` / `BREVO_SMTP_PASS` | Optional        | Required for Brevo password recovery mail.                            |
| `CRON_SECRET`                                                                 | Optional        | Required if using `/api/cron/auto-renew`.                             |
| `GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`                                   | Optional        | OAuth login.                                                          |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET`                                   | Optional        | OAuth login.                                                          |
| `NEXT_PUBLIC_THERMAL_PRINT_BRIDGE_URL`                                        | Optional        | Local thermal printer bridge URL, if used.                            |

## Vercel Project Settings

This repo includes `vercel.json` with:

- Install command: `npm ci`
- Build command: `npx prisma generate && npm run build`
- Output directory: `.next`

Vercel will run `postinstall` too, but the explicit Prisma generate step keeps deployments predictable.

## Database Setup

For first deployment:

```bash
npx prisma db push
npm run db:seed
```

For later production releases, prefer Prisma migrations:

```bash
npm run db:migrate
git add prisma/migrations
git commit -m "Add database migration"
```

Then deploy and run:

```bash
npm run db:migrate:prod
```

## File Uploads

The shared upload route now stores files in UploadThing when `UPLOADTHING_TOKEN` is configured. The database stores the returned URL, not a local file path.

Currently wired:

- Company logos
- System admin branding logo and favicon
- Product images
- Billing payment screenshots
- Generic document uploads through the shared route

For local development without `UPLOADTHING_TOKEN`, uploads fall back to data URLs so the app still works. Do not rely on that fallback in production.

## PDFs And Documents

Invoice, quotation, statement, and report PDFs should be generated on demand and streamed to the browser where possible.

Important production note: the app currently uses Puppeteer for some HTML-to-PDF routes. Vercel Functions do not provide a normal persistent local Chrome install, so production PDF rendering should be tested carefully with one of these approaches:

- Switch PDF routes to a Vercel-compatible Chromium package plus `puppeteer-core`.
- Use a browserless rendering provider.
- Keep simple PDF generation on the existing pure PDF generator where it is sufficient.

If you decide to store generated PDFs, upload the generated buffer to UploadThing or Vercel Blob and store the public/private URL in Postgres.

## ZATCA Notes

The local ZATCA SDK paths are intentionally blank in `.env.example` and `tools/zatca-sdk/` is ignored by Git. The Windows SDK/batch flow is a local validation tool, not a portable Vercel runtime dependency.

## Git And Vercel Flow

```bash
git init
git add .
git status
git commit -m "Prepare Cloud Daftar for deployment"
git branch -M main
git remote add origin https://github.com/YOUR_ACCOUNT/YOUR_REPO.git
git push -u origin main
```

After pushing, import the repository at Vercel, add the environment variables, and deploy.
